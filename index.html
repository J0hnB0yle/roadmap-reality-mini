Same shape I walked on screen yesterday, now runnable in the browser. Click Run to see the orchestration match commitments against the backlog, score gaps by revenue and renewal proximity, and draft reset language end-to-end.
Mock data · vanilla JS · no build step · same logic as run.py

Workflow · signal → triage → route

  1. Ingest — backlog, CRM call notes, renewal calendar
  2. Match — keyword anchors join AE commitments to backlog items (prod: model read)
  3. Classify — (tag, status) → CRITICAL / HIGH / MEDIUM / LOW
  4. Score — gap_score = arr ÷ days × 100
  5. Rank — sort by score, surface top three
  6. Draft — templated reset language per gap (prod: model gen)
  7. Emit — ranked list, top-3 cards, weekly digest

Stack

  • Demo: one HTML file, vanilla JS, no build, no deps
  • Hosting: GitHub Pages, static
  • Logic: direct port of run.py from the source mini-demo
  • Data: 3 embedded JS arrays (mirror of data/*.csv)
Production equivalent: n8n or Temporal orchestrator; Jira / Linear + Gong / Salesforce + CS-tool APIs as connectors; LLM call for commitment extraction and reset drafting; Slack post for the digest, scoped DMs per AE; severity weights in a config table so Ops can tune without a deploy. ~2-3 engineering-weeks plus 1-2 Ops weeks.

What's intentionally not in this build

  • Live integrations. Production swaps the embedded JS data for Jira, Gong/Salesforce, and CS-tool API calls.
  • Model-based commitment extraction. This demo uses keyword anchors. Production reads call-summary text with a model.
  • Per-AE Slack DMs. The mini-version emits one digest. Production routes drafted reset language to each AE individually.
  • Severity weights in code. In production they belong in a config table so Ops can tune without a deploy.
